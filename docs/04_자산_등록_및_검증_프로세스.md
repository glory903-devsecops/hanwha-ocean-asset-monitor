# 🔍 자산 등록 및 실시간 검증 가이드 (Verification Guide)
> **UI에서의 요청이 실제 데이터베이스에 반영되는 과정 확인하기**

사용자님께서 대시보드나 관리 페이지를 통해 자산을 추가했을 때, 해당 데이터가 시스템에 정상적으로 반영되었는지 확인하는 3가지 핵심 방법을 안내해 드립니다.

---

## 1. 정적 데모(Live Demo) vs 실제 시스템(Local Server)
먼저 중요한 차이점을 이해해야 합니다:
- **깃허브 라이브 데모**: 웹사이트 디자인을 보여주기 위한 **'프로토타입'**입니다. 브라우저에서 '등록 완료'를 눌러도 실제 서버가 없기 때문에 데이터가 저장되지 않습니다.
- **로컬 서버 (`127.0.0.1:8000`)**: 사용자님의 PC에서 실행되는 **'실제 시스템'**입니다. 여기서 등록한 자산은 `shipyard_assets.db` 파일에 영구 저장됩니다.

---

## 2. 자산 등록 확인 방법 (로컬 서버 기준)

### Method A: Swagger UI를 통한 데이터 조회 (권장)
가장 엔지니어다운 확인 방법입니다.
1. `python run_dev.py`를 실행하여 서버를 켭니다.
2. [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)에 접속합니다.
3. **`GET /api/v1/assets`** 섹션을 클릭합니다.
4. **[Try it out]** -> **[Execute]** 를 클릭합니다.
5. 아래 **Response body**에 새로 추가한 자산 정보가 JSON 형태로 나타나는지 확인합니다.

### Method B: 파이썬 검증 스크립트 실행
터미널에서 명령 한 줄로 DB에 저장된 자산 개수와 목록을 확인할 수 있습니다.
```powershell
# 가상 환경의 파이썬을 실용하여 실행 (권장)
.\venv\Scripts\python src/tests/test_results.py
```
*스크립트를 실행하면 현재 DB에 등록된 '총 자산 수'와 '장애 자산 현황'을 요약하여 출력해 줍니다.*

### Method C: DB 브라우저로 직접 확인
데이터베이스 파일(`shipyard_assets.db`)을 SQLite 전용 브라우저로 열어서 `assets` 테이블의 가장 마지막 행(Row)을 확인하면 실시간으로 추가된 내역을 볼 수 있습니다.

---

## 3. 자산 추가 프로세스 흐름 (Workflow)
채용 기획 단계 및 발표 시 다음 흐름을 설명하시면 매우 전문적인 인상을 줄 수 있습니다:

1. **Request (UI)**: 관리자가 `manage.html`에서 자산 정보를 입력하고 등록 버튼 클릭.
2. **Validation (FastAPI)**: 백엔드가 Pydantic 모델을 통해 데이터 형식(IP 주소, 자산 코드 등)이 올바른지 검사.
3. **Persistence (SQLAlchemy)**: 검증된 데이터를 DB에 안전하게 저장(Commit).
4. **Audit (Logging)**: "누가, 언제, 어떤 자산을 추가했는지" 로그 파일에 기록.
5. **Update (Dashboard)**: 다음 대시보드 새로고침 시 변경된 데이터가 반영되어 출력.
