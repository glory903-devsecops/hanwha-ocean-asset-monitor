# ⚓ 조선소 IT/OT 통합 자산 모니터링 시스템 기획안
> **한화오션 AX(AI Transformation) 전략 연계 프로젝트**

---

## 1. 프로젝트 개요
### 1.1 배경 및 목적
한화오션의 **'스마트 야드 2030'** 비전은 생산 현장 자동화율 70% 달성을 목표로 합니다. 본 프로젝트는 기하급수적으로 증가하는 IT 자산과 OT(PLC, 센서, 로봇) 장비를 단일 창구에서 관리하여 **운영 효율성을 극대화**하는 것을 목적으로 합니다.

### 1.2 핵심 목표
* **통합 가시성(Visibility)**: 분산된 IT/OT 장비의 실시간 상태 통합 조회.
* **다운타임 최소화**: 이상 탐지 엔진을 통한 선제적 대응 체계 구축.
* **라이프사이클 자동화**: 도입부터 폐기까지 전 과정을 시스템적으로 추적.

---

## 2. 한화오션 AX 전략 연계
### 2.1 스마트 야드 비전 접점
* **연결화**: 수천 대의 IoT 장비와 IT 인프라를 하나의 아키텍처로 통합.
* **지능화**: 축적된 데이터를 바탕으로 AI 기반 예방 정비(CBM)로 나아가기 위한 데이터 파운데이션 구축.

### 2.2 JD(Job Description) 매핑
| JD 요구 역량 | 본 프로젝트 반영 내용 |
| :--- | :--- |
| 전사 시스템 개발 및 운영 | FastAPI & React 기반 3-Tier 아키텍처 구현 |
| 업무 자동화(RPA) 시스템 개발 | 장비 장애/EOL 도래 시 자동 티켓 생성 모듈 |
| IT 자산 및 인프라 관리 | CMDB 기반 자산 라이프사이클 관리 시스템 |

---

## 3. 시스템 아키텍처
### 3.1 기술 스택
* **Backend**: Python (FastAPI), SQLAlchemy, Pydantic
* **Frontend**: Vanilla JS (Premium UI), ECharts (시각화)
* **Database**: PostgreSQL (CMDB), Redis (실시간 알림)
* **Protocol**: MQTT (IoT), SNMP (Network Infrastructure)

### 3.2 핵심 모듈
1. **CMDB 모듈**: 자산 간 의존성 매핑 및 관계 관리.
2. **이상 탐지 엔진**: 규칙 기반 및 통계적 이상치 탐지 로직 적용.
3. **RPA Webhook**: 외부 플랫폼 연동을 위한 인터페이스 제공.

---

## 4. 데이터베이스 설계 (CMDB)
### 4.1 핵심 테이블 구조
* **Assets**: 자산 고유 정보, 상태, 라이프사이클 단계, 메타데이터(JSONB).
* **Metrics**: CPU, 메모리, 온도 등 실시간 수집 메트릭 (시계열 데이터).
* **Alerts**: 이상 탐지 이력 및 조치 상태 추적.

---

## 5. 실무 경험의 반영
본 기획은 **GE Korea IT Support Engineer**로서 직접 경험한 ServiceNow 기반 자산관리(ITAM) 경험을 바탕으로 설계되었습니다. 실제 현장의 Pain Point인 '파편화된 자산 정보'와 '사후 대응 중심의 운영'을 해결하는 데 초점을 맞추었습니다.
