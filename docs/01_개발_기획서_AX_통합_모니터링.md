# **1\. 프로젝트 개요**

## **1.1 프로젝트 배경**

한화오션은 거제 사업장에 약 3,000억 원을 투입하여 2030년까지 생산 현장 자동화율 70%를 달성하는 '스마트 야드' 전략을 추진하고 있습니다. 드론과 IoT 센서를 활용한 실시간 현장 모니터링, 80대 이상의 용접·가공 로봇 투입, AI 기반 공정 최적화 등이 이미 현장에 적용되고 있으며, 이러한 디지털 전환은 미국 필리조선소로의 확산까지 계획되어 있습니다.

스마트 야드의 고도화가 진행될수록, 조선소 내 관리해야 할 IT 자산과 IoT 장비의 수는 기하급수적으로 증가합니다. PC, 서버, 네트워크 장비 등 전통적 IT 자산에 더해 PLC, 센서, 로봇 컨트롤러 등 OT(Operational Technology) 장비까지 통합적으로 모니터링하고 관리해야 하는 필요성이 대두되고 있습니다.

현재 대부분의 조선소에서는 IT 자산과 IoT 장비가 별도의 시스템으로 관리되고 있어, 장비 장애 시 원인 파악에 시간이 소요되고, 자산 라이프사이클 관리가 분산되어 비효율이 발생합니다. 본 프로젝트는 이러한 문제를 해결하기 위해, IT 자산과 IoT 장비를 하나의 웹 기반 대시보드에서 통합 모니터링하는 시스템을 설계·개발합니다.

## **1.2 프로젝트 목적**

* 조선소 내 IT 자산(PC, 서버, 네트워크 장비)과 생산 현장 IoT 장비(센서, PLC, 로봇 컨트롤러)의 통합 가시성(Visibility) 확보

* 실시간 장비 상태 모니터링 및 이상 탐지를 통한 다운타임 최소화

* 자산 라이프사이클 관리(도입→운영→교체)의 자동화 및 체계화

* RPA 연동을 통한 장비 교체·구매 요청 프로세스 자동화

* 데이터 시각화 대시보드를 통한 의사결정 지원

| 구분 | 기대 효과 | 상세 내용 |
| ----- | ----- | ----- |

# **2\. 한화오션 AX 전략과의 연계**

## **2.1 스마트 야드 비전과 프로젝트 접점**

한화오션 AX사업부는 인공지능 전환(AI Transformation)을 주도하며, '연결화·자동화·지능화'를 핵심 키워드로 스마트 야드를 구축하고 있습니다. 2021년 조선업계 최초로 설립된 디지털 생산센터는 드론·IoT 센서 기반의 실시간 모니터링을 운영하고 있으며, AI 드론 흘수 계측 시스템은 기존 2시간 소요 작업을 30분 이내로 단축시키는 성과를 거두었습니다.

본 프로젝트는 스마트 야드의 '연결화' 단계에서 핵심 인프라 역할을 합니다. 생산 현장에 배치된 수백\~수천 대의 IoT 장비와 IT 자산의 상태를 통합 모니터링함으로써, '자동화'와 '지능화' 단계로 나아가기 위한 데이터 기반(Data Foundation)을 구축합니다.

또한, 한화오션이 개발 중인 통합 MRO 플랫폼과의 연계 가능성도 고려합니다. IT 자산·IoT 장비의 정비 이력과 상태 데이터를 MRO 플랫폼에 제공함으로써, 설계·건조·운영·정비에 이르는 '디지털 스레드(Digital Thread)' 구현에 기여할 수 있습니다.

# **3\. 시스템 아키텍처**

## **3.1 전체 시스템 구성도**

본 시스템은 크게 데이터 수집 계층(Collection Layer), 처리·분석 계층(Processing Layer), 표현 계층(Presentation Layer)의 3-Tier 아키텍처로 구성됩니다.

| ┌─────────────────────────────────────────────────────┐ │              \[ Presentation Layer \]                 │ │   React \+ ECharts 대시보드  │  관리자 콘솔         │ ├─────────────────────────────────────────────────────┤ │               \[ API Gateway \+ JWT Auth \]            │ ├─────────────────────────────────────────────────────┤ │              \[ Processing Layer \]                   │ │  FastAPI Backend                                    │ │  ┌──────────┐ ┌──────────┐ ┌────────────────────┐  │ │  │ CMDB     │ │ 이상탐지 │ │ RPA 연동 Engine   │  │ │  │ Module   │ │ Engine   │ │ (자동 티켓 생성)   │  │ │  └──────────┘ └──────────┘ └────────────────────┘  │ │  ┌──────────────────────┐  ┌────────────────────┐  │ │  │ Redis (실시간 알림)  │  │ PostgreSQL (DB)    │  │ │  └──────────────────────┘  └────────────────────┘  │ ├─────────────────────────────────────────────────────┤ │              \[ Collection Layer \]                   │ │  ┌──────────┐ ┌──────────┐ ┌────────────────────┐  │ │  │ SNMP     │ │ MQTT     │ │ REST API 수집기    │  │ │  │ Collector │ │ Broker   │ │ (외부 시스템 연동) │  │ │  └────┬─────┘ └────┬─────┘ └─────────┬──────────┘  │ └────────┼────────────┼─────────────────┼─────────────┘     네트워크 장비    IoT 센서/PLC      ERP/MRO 시스템     서버/스토리지    로봇 컨트롤러     ServiceNow 등 |
| :---- |

## **3.2 주요 모듈 설명**

**① IT 자산 관리 모듈 (CMDB)**

ServiceNow CMDB와 유사한 구조로 설계하며, 조선소 환경에 최적화합니다. 모든 IT 자산(PC, 서버, 네트워크 장비, 프린터 등)의 메타데이터를 관리하고, 자산 도입부터 폐기까지의 전체 라이프사이클을 추적합니다. 자산 간 의존성 맵(Dependency Map)을 통해 특정 장비 장애 시 영향 범위를 즉시 파악할 수 있습니다.

**② IoT 장비 데이터 수집 모듈**

MQTT 프로토콜을 통해 생산 현장의 IoT 센서, PLC, 로봇 컨트롤러에서 발생하는 데이터를 실시간으로 수집합니다. SNMP를 통해 네트워크 장비(스위치, 라우터, 방화벽)의 상태 정보를 주기적으로 폴링합니다. 수집된 데이터는 정규화(Normalization)를 거쳐 통합 데이터베이스에 저장됩니다.

**③ 실시간 모니터링 대시보드**

React \+ Apache ECharts 기반의 웹 대시보드로, 자산 현황 요약, 장비별 가동률, 네트워크 트래픽, 센서 데이터 트렌드를 시각화합니다. 조선소 영역별(도크, 안벽, 절단장, 조립장 등) 필터링과 장비 유형별 뷰를 제공하며, WebSocket을 통한 실시간 데이터 갱신을 지원합니다.

**④ 이상 탐지 및 알림 엔진**

규칙 기반(Rule-based) 이상 탐지를 1차로 적용하고, 통계적 이상치 탐지(Z-score, IQR)를 2차로 적용합니다. 예: CPU 사용률 90% 초과 5분 지속, IoT 센서 응답 없음 3분 이상, 네트워크 패킷 로스율 5% 초과 등. Redis Pub/Sub를 통해 실시간 알림을 전달하며, 이메일/SMS/Slack 등 다채널 알림을 지원합니다.

**⑤ RPA 연동 모듈 (자동 티켓 생성)**

장비의 예상 수명(EOL)이 임박하거나, 특정 장비에 반복적 장애가 발생할 때 자동으로 구매 요청/교체 요청 티켓을 생성합니다. API 기반으로 외부 RPA 플랫폼(UiPath, Power Automate 등)과 연동할 수 있는 Webhook 인터페이스를 제공합니다. 한화오션 AX사업부가 추진하는 행정·문서 업무 자동화와 동일한 접근 방식을 적용합니다.

**⑥ 보안 모듈 (인증/인가/로깅)**

JWT(JSON Web Token) 기반 인증과 RBAC(Role-Based Access Control) 기반 인가를 구현합니다. 역할은 시스템 관리자, IT 자산 관리자, 현장 모니터링 담당자, 뷰어(읽기 전용) 등으로 구분합니다. 모든 API 요청에 대한 감사 로그(Audit Log)를 기록하며, IP 기반 접근 제한과 Rate Limiting을 적용합니다.

# **4\. 기술 스택 상세**

## **4.1 기술 선택 근거**

각 기술은 한화오션 AX사업부의 기술 환경과 JD 요구사항을 고려하여 선택하였습니다. 특히, 1인 개발 프로젝트의 생산성과 실무 환경에서의 확장성을 동시에 충족하는 것을 기준으로 삼았습니다.

| 영역 | 기술 | 선택 이유 |
| ----- | ----- | ----- |
| **Backend** | Python (FastAPI) | JD 우대사항 'Python' 부합. 비동기 처리로 IoT 실시간 데이터 수집에 최적. 자동 API 문서(Swagger) 생성 |
| **Frontend** | React \+ ECharts | 컴포넌트 기반 대시보드 개발에 최적. ECharts는 대용량 데이터 시각화 성능이 뛰어나며, JD 우대사항 '데이터 시각화' 부합 |
| **Database** | PostgreSQL | JSONB 타입으로 다양한 IoT 메타데이터 유연하게 저장. 시계열 확장(TimescaleDB)이 용이하여 센서 데이터 관리에 적합 |
| **메시지 큐** | Redis | Pub/Sub 기반 실시간 알림 처리. 캐싱을 통한 대시보드 응답 속도 최적화. 경량이면서 고성능 |
| **컨테이너** | Docker \+ Compose | 개발·운영 환경 일관성 보장. 1인 개발 시 로컬에서 전체 스택 구동 가능. 한화오션 클라우드 전략과 부합 |
| **인프라** | AWS (EC2, RDS, S3) | JD 우대사항 '클라우드' 부합. 개발 단계는 AWS, 실제 배포는 On-premise 전환 가능한 구조로 설계 |
| **데이터 수집** | SNMP / MQTT | SNMP는 네트워크 장비 표준 프로토콜. MQTT는 IoT 환경의 경량 메시징 표준으로 조선소 현장 적용에 최적 |
| **보안** | JWT \+ RBAC | Stateless 인증으로 확장성 확보. 역할 기반 접근 제어로 조선소 부서별 권한 분리. 정보보호 대학원 전공 역량 반영 |
| **IaC** | Terraform | 인프라를 코드로 관리하여 재현 가능한 환경 구성. 필리조선소 등 글로벌 확장 시 동일 인프라 복제 용이 |

# **5\. 주요 기능 명세**

**기능 1: 통합 자산 등록 및 조회 (CMDB)**

IT 자산과 IoT 장비를 단일 인터페이스에서 등록·조회·수정·삭제합니다. 자산 유형(서버, PC, 스위치, 센서, PLC 등)별 커스텀 속성을 정의할 수 있으며, 바코드/QR 코드 스캔을 통한 현장 자산 등록을 지원합니다.

**해결하는 실무 문제:** *기존: IT 자산은 엑셀, IoT 장비는 별도 시스템으로 관리 → 데이터 불일치, 이중 관리 공수 발생*

**기능 2: 자산 라이프사이클 관리**

도입(Procurement) → 배치(Deployment) → 운영(Operation) → 유지보수(Maintenance) → 폐기(Disposal)의 전체 단계를 추적합니다. 각 단계별 상태 변경 이력과 담당자를 자동 기록하며, EOL(End of Life) 임박 자산에 대한 사전 알림을 제공합니다.

**해결하는 실무 문제:** *기존: 장비 교체 시기를 개인 기억이나 엑셀에 의존 → 교체 지연으로 장애 발생 위험*

**기능 3: 실시간 장비 상태 모니터링**

SNMP/MQTT를 통해 수집된 장비 상태 데이터(CPU/메모리 사용률, 온도, 네트워크 트래픽, 센서 값 등)를 실시간으로 시각화합니다. 조선소 영역별 맵 뷰와 리스트 뷰를 모두 제공하며, 장비별 상세 드릴다운이 가능합니다.

**해결하는 실무 문제:** *기존: 장비 상태 확인을 위해 현장 방문 또는 개별 관리 콘솔 접속 필요 → 대응 지연*

**기능 4: 이상 탐지 및 자동 알림**

임계값 기반 규칙 엔진과 통계적 이상치 탐지를 결합하여, 장비 이상을 조기에 감지합니다. 알림은 심각도(Critical/Warning/Info)에 따라 차등 발송되며, 에스컬레이션 정책(미확인 시 상위 담당자에게 자동 전달)을 적용합니다.

**해결하는 실무 문제:** *기존: 장비 장애를 사용자 신고로 인지 → 사후 대응 중심의 비효율적 운영*

**기능 5: RPA 연동 자동 티켓 생성**

EOL 임박, 반복 장애 발생, 라이선스 만료 등 사전 정의된 조건 충족 시, 외부 RPA 플랫폼으로 Webhook을 전송하여 구매 요청/교체 요청 티켓을 자동 생성합니다. 티켓 생성 이력과 처리 현황을 대시보드에서 추적할 수 있습니다.

**해결하는 실무 문제:** *기존: 장비 교체 요청을 수작업으로 작성·결재 → 프로세스 지연 및 누락 발생*

**기능 6: 네트워크 토폴로지 시각화**

조선소 네트워크의 물리적·논리적 토폴로지를 자동으로 탐색(Discovery)하고 시각화합니다. 각 노드의 실시간 상태(정상/경고/장애)를 색상으로 표현하며, 링크별 트래픽 현황을 시각화하여 병목 구간을 식별합니다.

**해결하는 실무 문제:** *기존: 네트워크 구성도를 수동으로 관리(Visio 등) → 실제 환경과 불일치*

**기능 7: 리포트 및 통계 분석**

일별/주별/월별 자산 가동률, 장애 발생 빈도, 평균 복구 시간(MTTR), 장비 유형별 고장률 등 핵심 KPI를 자동 산출합니다. Power BI 연동을 위한 데이터 내보내기(CSV/JSON) 기능을 제공합니다.

**해결하는 실무 문제:** *기존: 장비 관련 통계를 수작업으로 집계 → 보고서 작성에 과다한 시간 소요*

**기능 8: 감사 로그 및 보안 대시보드**

모든 사용자 행위(로그인, 자산 수정, 설정 변경 등)에 대한 감사 로그를 자동 기록합니다. 보안 대시보드에서 비정상 접근 시도, 권한 변경 이력, API 호출 패턴을 모니터링할 수 있습니다.

**해결하는 실무 문제:** *기존: 시스템 접근 이력 미관리 → 보안 사고 발생 시 원인 추적 불가*

# **9\. 기존 경력과의 연결**

## **9.1 GE Korea IT 자산관리 경험의 적용**

GE Korea에서 3년 5개월간 IT Support Engineer로 근무하며 ServiceNow 기반의 IT 자산관리(ITAM)를 수행한 경험은 본 프로젝트의 핵심 도메인 지식입니다. ServiceNow CMDB의 구조와 운영 방식을 직접 경험했기에, 자산 등록·분류·관계 매핑·변경 관리의 Best Practice를 시스템 설계에 반영할 수 있습니다.

* **자산 분류 체계:** GE의 글로벌 IT 자산 분류 체계 경험을 바탕으로, 조선소 환경에 맞는 IT/OT 통합 분류 체계를 설계하였습니다.

* **CMDB 데이터 모델:** ServiceNow CI(Configuration Item) 구조를 참고하되, IoT 장비의 특성(센서 타입, 통신 프로토콜, 펌웨어 버전 등)을 포함하는 확장 스키마를 설계하였습니다.

* **인시던트-자산 연계:** ServiceNow에서 인시던트와 CI를 연결하여 영향 분석을 수행했던 경험을 반영하여, 알림(Alert)과 자산을 자동으로 연계하는 로직을 구현합니다.

* **네트워크/인프라 운영:** 네트워크 장비 관리 및 인프라 운영 경험이 SNMP 기반 모니터링 설계와 네트워크 토폴로지 시각화 기능의 기반이 되었습니다.

## **9.2 정보보호 대학원/보안 부트캠프의 반영**

현재 재학 중인 정보보호 대학원과 사이버 보안 부트캠프(클라우드 보안 시스템 개발) 수료 경험은 본 프로젝트의 보안 설계에 직접적으로 반영되었습니다.

* **인증/인가 설계:** JWT 기반 Stateless 인증, RBAC 기반 세분화된 권한 관리, 토큰 갱신 및 블랙리스트 메커니즘 등 보안 전공자 수준의 인증 체계를 구현합니다.

* **감사 로그:** 모든 사용자 행위에 대한 감사 로그 자동 기록 및 변조 방지(immutable log) 설계는 정보보호 관점에서 필수적인 요소입니다.

* **클라우드 보안:** AWS 인프라 설계 시 VPC 분리, Security Group 설정, IAM 최소 권한 원칙 적용 등 클라우드 보안 부트캠프에서 학습한 내용을 적용합니다.

* **OT 보안 고려:** IoT 장비와 IT 네트워크 간의 네트워크 세그먼테이션, MQTT 통신 TLS 암호화 등 OT 보안 특수성을 반영합니다.

## **9.3 운영 경험에서 개발 역량으로**

본 프로젝트는 단순한 기술 시연이 아닌, 3년 이상의 IT 운영 현장에서 직접 느꼈던 문제점과 개선 필요성에서 출발한 기획입니다.

IT Support Engineer로서 '장비 장애가 왜 이렇게 늦게 감지되는가?', 'IT 자산과 IoT 장비를 왜 따로 관리해야 하는가?', '장비 교체 요청을 왜 수작업으로 해야 하는가?'라는 질문들을 현장에서 수없이 마주했습니다. 이 프로젝트는 그 질문들에 대한 기술적 답변입니다.

운영 경험이 있는 개발자는 현장의 실제 Pain Point를 정확히 이해하고, 실무에서 바로 쓸 수 있는 시스템을 설계할 수 있습니다. 이것이 '운영에서 개발로'의 전환이 가지는 차별화된 가치입니다.

# **10\. 향후 확장 계획**

## **Phase 2: AI 예측 정비 연동 (3\~6개월)**

* 축적된 장비 메트릭 데이터를 기반으로 머신러닝 예측 모델을 개발하여, 장비 고장을 사전에 예측하는 CBM(Condition-Based Maintenance) 기능을 추가합니다.

* 한화오션이 추진 중인 AI 기반 예측 정비 전략과 연계하며, 센서 데이터의 시계열 분석(LSTM, Prophet 등)을 적용합니다.

* 예측 결과를 RPA 모듈과 연동하여, 고장 예측 시 선제적 정비 티켓 자동 생성을 구현합니다.

## **Phase 3: 디지털 트윈 연계 (6\~12개월)**

* 조선소 레이아웃의 3D 모델과 연동하여, 디지털 트윈 환경에서 IT 자산 및 IoT 장비의 위치와 상태를 실시간 시각화합니다.

* 한화오션 디지털 생산센터의 S YARD 플랫폼과 데이터 연동 인터페이스를 구현하여, 생산 현장 모니터링의 통합 뷰를 제공합니다.

* WebGL 기반 3D 시각화(Three.js)를 통한 직관적 공간 기반 모니터링 경험을 제공합니다.

## **Phase 4: 필리조선소(미국) 글로벌 확장 (12개월+)**

* 한화오션의 미국 필리조선소 스마트 야드 전환에 맞춰, 본 시스템을 글로벌 환경으로 확장합니다.

* 다국어(한국어/영어) 지원, 타임존 처리, 거제↔필라델피아 간 데이터 동기화 등 글로벌 운영 요구사항을 충족합니다.

* Terraform 기반 IaC로 동일한 인프라를 AWS 미국 리전에 복제 배포하며, 멀티 리전 아키텍처를 적용합니다.

* 한화오션이 계획 중인 필리조선소 총 50억 달러 투자 및 연간 건조 능력 20척 확대 전략과 연계한 확장형 인프라를 설계합니다.
