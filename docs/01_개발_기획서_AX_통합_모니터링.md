# 조선소 IT/OT 통합 자산 모니터링 시스템 기술 기획서
> **AI Transformation(AX) 기반 스마트 야드 운영 자동화 프로젝트**

---

## 1. 프로젝트 목적 및 배경
한화오션의 **'스마트 야드 2030'** 비전 달성을 위해, 생산 고도화의 핵심 기반인 IT/OT 통합 자산 관리 체계를 구축합니다. 폭발적으로 증가하는 현장 장비(PLC, 로봇, 센서)와 이를 제어하는 인프라를 단일 플랫폼에서 관리함으로써 운영 가시성을 확보하는 것이 본 프로젝트의 최우선 목적입니다.

### 핵심 시스템 가치
- **통합 가시성(Unified Visibility)**: 이기종 장비 간의 의존성 파악 및 실시간 상태 모니터링.
- **운영 안정성(Operational Stability)**: 이상 징후 조기 발견을 통한 생산 중단 시간(Downtime) 최소화.
- **데이터 기반 의사결정(Data-Driven Decision)**: 축적된 장비 메트릭을 바탕으로 한 예방 정비 기초 데이터 확보.

---

## 2. 주요 시스템 요건 (System Requirements)
스마트 야드 현장의 특수성을 고려하여 다음과 같은 핵심 요건을 설계에 반영했습니다:

- **엔터프라이즈급 인프라 관리**: 데이터 센터 서버부터 현장 네트워크 스위치까지의 토폴로지 관리.
- **산업용 프로토콜 수용**: MQTT, SNMP 등 제조/설비 분야 표준 프로토콜 연동 레이어 확보.
- **자산 라이프사이클 자동화**: 도입, 배포, 유지보수, 폐기에 이르는 전 과정을 CMDB(구성 관리 데이터베이스) 기반으로 이력 관리.

---

## 3. 시스템 아키텍처 (Technical Architecture)

### 3.1 기술 스택 (Tech Stack)
- **Backend**: Python (FastAPI) - 고성능 비동기 API 처리.
- **Frontend**: Vanilla JS (Premium UI), ECharts - 대용량 데이터 전용 시각화.
- **Database**: PostgreSQL (CMDB) - 관계형 및 JSONB를 활용한 유연한 자산 모델링.
- **Infra**: GitHub Actions - 지속적 통합 및 자동 배포(CI/CD) 엔진.

### 3.2 핵심 모듈 구조
1. **CMDB core**: 자산 간의 계층 구조 및 의존성 매핑 엔진.
2. **Alert Engine**: 실시간 메트릭 분석을 통한 임계치 기반 이상 탐지 모듈.
3. **API Gateway**: 외부 시스템(RPA 등)과의 유기적 연동을 위한 인터페이스.

---

## 4. 데이터베이스 및 인프라 설계
### 4.1 데이터 모델링 (CMDB Schema)
- **Asset Master**: 자산 고유 정보, 상태 코드, 물리적 위치 및 라이프사이클 단계 정보.
- **Metric Repository**: CPU 부하, 온도, 진동 등 현장 기기의 시계열 메트릭 저장소.
- **Audit Trails**: 데이터 변경 이력 및 시스템 조작 로그를 통한 무결성 보장.

### 4.2 배포 및 검증 전략
본 프로젝트는 **DevSecOps** 원칙에 따라 모든 코드 변경 사항이 자동화된 테스트를 거쳐 배포됩니다. 이는 시스템의 신뢰성을 보장하며 현장 적용 시 리스크를 최소화하는 핵심 전략입니다.
