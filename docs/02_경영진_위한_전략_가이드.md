# 🏆 경영진을 위한 전략 가이드: 한화오션 AX 구현 사례
> **대상: IT 부문장(CIO/CTO), 채용 담당자 (AX 사업부)**

### 1. "왜 이 시스템이 필요한가?" (ROI 관점)
전통적인 조선소 환경에서는 IT 자산(서버, PC)과 생산 현장의 OT 장비(센서, 로봇)가 별도로 관리되어 왔습니다. 이러한 파편화는 다음과 같은 고질적인 문제를 야기합니다:
- **장애 대응 지연**: 생산 현장의 PLC나 로봇 컨트롤러가 멈춰도 IT 부서가 이를 실시간으로 인지하지 못합니다.
- **불확실한 자산 현황**: 수동 엑셀 관리는 데이터 누락과 구매 오류로 이어집니다.
- **운영 공백**: 야드 전체의 디지털 상태(Digital Health)를 확인할 수 있는 단일 창구가 없습니다.

**본 솔루션은 IT와 OT를 잇는 360도 가시성을 제공합니다.** 단순한 대시보드를 넘어, 데이터 센터의 서버와 생산 라인의 로봇 간의 상관관계를 이해하는 **중앙 집중형 커맨드 센터(Centralized Command Center)**의 역할을 수행합니다.

### 2. 한화오션 스마트 야드 2030 전략과의 연계
- **연결된 야드 (Connected Yard)**: MQTT(IoT)와 SNMP(IT)를 하나의 아키텍처로 통합.
- **자동화된 야드 (Automated Yard)**: RPA 연동을 통해 장애 감지부터 부품 조달까지 "개입 없는 자동화" 구현의 초석.
- **지능형 야드 (Autonomous Yard)**: 본 시스템이 축적한 데이터는 향후 **AI 기반 예방 정비(CBM)** 구축을 위한 핵심 기반 데이터가 됩니다.

### 3. 엔지니어링 전문성 및 DevSecOps
- **보안 중심 설계**: 정보보호 전공 역량을 살려 JWT 기반 인증 및 RBAC(역할 기반 접근 제어)를 적용했습니다.
- **확장성**: 수천 대의 조선소 자산을 수용할 수 있는 3계층(3-Tier) 웹 아키텍처로 설계되었습니다.
- **무결성**: 데이터 조작 방지를 위한 감사 로그(Audit Log)와 정교한 SQLAlchemy 데이터 모델을 적용했습니다.
