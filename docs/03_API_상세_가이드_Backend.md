# 🖥️ 백엔드 API 시스템 가이드 (Backend Technical Guide)
> **FastAPI 기반 IT/OT 통합 자산 관리 브레인**

본 문서는 프로젝트의 핵심 로직인 백엔드 API(`127.0.0.1:8000/docs`)를 이해하고 활용하기 위한 가이드입니다. 

---

## 1. FastAPI & Swagger UI란?
이 프로젝트의 백엔드는 **FastAPI** 프레임워크를 사용합니다. 
로컬에서 서버를 실행한 후 `/docs` 경로로 접속하면 나타나는 화면은 **Swagger UI**라고 불리며, 다음과 같은 기능을 제공합니다:
- **자동 문서화**: 코드를 수정하면 실시간으로 API 명세가 업데이트됩니다.
- **인터랙티브 테스트**: 별도의 도구(Postman 등) 없이도 브라우저에서 직접 데이터를 조회하거나 전송할 수 있습니다.

---

## 2. 주요 API 엔드포인트 상세

### 📡 [GET] `/api/v1/assets` (자산 목록 조회)
시스템에 등록된 모든 IT 및 OT 자산의 목록을 가져오는 핵심 API입니다.
- **출력 데이터**: 자산명, 상태(정상/경고), IP 주소, 메트릭(CPU/온도) 등.
- **실무 활용**: 대시보드의 전체 현황판을 구성하는 원천 데이터가 됩니다.

### 🏥 [GET] `/health` (시스템 상태 체크)
서버와 데이터베이스가 정상적으로 연결되어 있는지 확인합니다.
- **출력 데이터**: `{"status": "ok", "version": "1.0.0"}`

---

## 3. 데이터 구조 (Data Schema)
백엔드는 **Pydantic** 모델을 사용하여 데이터의 형식을 엄격하게 관리합니다. 이는 잘못된 데이터가 유입되는 것을 방지하여 시스템의 **안전성**을 높입니다.

**예시: 자산 데이터 구조 (Asset Schema)**
```json
{
  "asset_id": "SRV-GJ-001",
  "name": "거제 메인 서버",
  "type": "SERVER",
  "status": "active",
  "metrics": {
    "cpu_usage": 45.2,
    "temperature": 38.5
  }
}
```

---

## 4. 백엔드 설계의 기술적 강점 (Portfolio Point)
채용담당자에게 어필할 수 있는 기술적 포인트들입니다:
1. **비동기 처리(Async)**: 수천 개의 IoT 센서 데이터를 병목 현상 없이 동시에 수집할 수 있는 구조입니다.
2. **SQLAlchemy ORM**: 데이터베이스 관리를 객체 지향적으로 처리하여 유지보수성을 극대화했습니다.
3. **데이터 무결성**: 모든 자산 관리는 트랜잭션을 통해 데이터의 유실이나 중복 없이 안전하게 처리됩니다.

---

## 5. 프론트엔드(데모)와의 연결
현재 보시는 **Live Demo(GitHub Pages)**는 이 백엔드 API가 실제로 작동했을 때 보여질 결과를 미리 시각화한 것입니다. 실제 운영 단계에서는 이 API 문서에 정의된 엔드포인트를 통해 데이터를 주고받으며 화면이 갱신됩니다.

> [!NOTE]
> **백엔드 코드는 `src/backend/` 디렉터리에서 확인할 수 있습니다.**
